# 工作台 V1 试跑 / 演示路径清单 v1

更新时间：2026-03-31

## 1. 目标

这份清单用于统一说明：
- 工作台 V1 应该怎么演示
- 工作台 V1 应该怎么试跑
- 演示时重点验证哪些链路
- 当前阶段哪些属于“演示通过即可”，哪些还不是本阶段目标

---

## 2. 当前演示口径

当前工作台 V1 的推荐演示口径是：

> 从一个真实会话进入工作台后，可以先看到顶部入口摘要，再下钻到右侧判断层和执行层，最后回到左侧消息与审计证据链，完成“判断 → 执行 → 回流 → 证据确认”的最小闭环演示。

注意：
当前演示目标是证明：
- 页面结构已经可用
- 主次关系已成立
- 动作反馈闭环可跑

不是证明：
- 产品终态已完成
- 所有业务场景已覆盖
- 所有字段都已产品化

---

## 3. 标准演示路径

### 步骤 1：进入一个真实会话详情页
演示点：
- 能正常进入工作台
- 页面不是接口拼盘，而是有明确结构

观察重点：
- 顶部是否先给出总判断
- 左右分层是否清楚

通过标准：
- 用户进入后，3-5 秒内能理解当前页面在讲什么

### 步骤 2：查看顶部入口摘要
演示点：
- 先看“这是谁 / 当前状态 / 为什么需要处理 / 建议下一步”

观察重点：
- 是否能快速建立业务判断
- 是否减少先扫大量细节的成本

通过标准：
- 不看左右明细，也能大致说出当前会话情况

### 步骤 3：查看右侧深一层判断
演示点：
- 查看 insight / D4
- 查看 business-feedback
- 查看更细一层判断与建议

观察重点：
- 右侧第一块是否承接顶部摘要
- 是否能回答“为什么这么判断”

通过标准：
- 能从判断层读出当前建议与业务风险

### 步骤 4：进入右侧执行层
演示点：
- 查看 pending actions
- 查看 action history
- 选择一个动作做最小反馈提交

建议操作：
- 选择一个待处理动作
- 选择状态
- 选择 feedbackType
- 填备注（可选）
- 提交反馈

观察重点：
- 执行层是否独立清楚
- 反馈动作是否容易理解
- 提交后是否能立即看到回流结果

通过标准：
- 可以顺利完成一次动作反馈提交
- 页面能看到本次反馈回流

### 步骤 5：回到左侧最近消息
演示点：
- 查看最近消息
- 使用“只看可判断 / 文本 / 最近5条”等过滤方式

观察重点：
- 左侧是否像上下文与证据区，而不是另一个摘要区
- 是否能支撑刚才的判断与执行动作

通过标准：
- 左侧消息足以作为判断证据来源

### 步骤 6：查看底部治理证据链 / 审计联动
演示点：
- 查看最近审计记录
- 确认治理动作与落库证据

观察重点：
- 是否能形成“消息上下文 → 判断 → 执行 → 审计证据”的闭环

通过标准：
- 能清楚说明本次动作不是只改了界面，而是有治理记录与证据链承接

---

## 4. 试跑时建议重点验证的链路

### 链路 A：入口摘要链
验证：
- 顶部入口摘要是否稳定
- 不同会话下是否都能给出基本可读结果

### 链路 B：判断层链
验证：
- insight 生成与展示是否稳定
- business-feedback 是否能正常显示
- 两者是否能共同支撑业务判断

### 链路 C：执行反馈链
验证：
- pending actions 能读取
- 最小反馈表单能提交
- feedbackType 校验正常
- 本次反馈回流能看到
- action history 能展示结果

### 链路 D：证据链
验证：
- 最近消息可读取
- 最近治理动作可读取
- 审计联动可展示
- 能形成完整证据链说明

---

## 5. 当前阶段的通过标准

当前阶段可认为“试跑通过”的标准是：

- [x] 能进入工作台详情页
- [x] 顶部入口摘要可读
- [x] 右侧判断层可读
- [x] 右侧执行层可操作
- [x] 可完成一次最小反馈提交
- [x] 可看到本次反馈回流
- [x] 左侧消息与上下文可支撑判断
- [x] 底部审计联动可作为证据链展示

只要以上链路能稳定跑通，就可以表述为：

> 工作台 V1 已达到可演示、可试跑水位。

---

## 6. 当前阶段明确不要求的内容

当前试跑不要求：
- 所有业务类型全覆盖
- 所有字段都完全产品化
- 完整权限系统
- 更复杂统计分析页
- 更复杂多角色协同页
- 完整终态交互细节

---

## 7. 推荐演示顺序（最短版）

推荐最短演示顺序：

1. 打开会话详情页
2. 看顶部入口摘要
3. 看右侧深一层判断
4. 做一次最小反馈提交
5. 看反馈回流
6. 看左侧最近消息
7. 看底部治理证据链 / 审计联动

一句话解释：

> 先看结论，再看判断，再做执行，再回到证据。

---

## 8. 当前阶段结论

当前更适合的阶段性结论是：

> 工作台 V1 已具备标准演示路径与最小试跑闭环，可以进入实际试跑与演示验证阶段。
