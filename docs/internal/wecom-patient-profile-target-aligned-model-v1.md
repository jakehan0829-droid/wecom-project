# 患者档案模型（目标对齐版）v1

## 一、目标定义

患者档案不是一轮互动的“需求/顾虑卡片”，而是围绕真实业务目标持续沉淀的 **患者画像 + 问题画像 + 跟进画像**。

信息来源必须同时包括：
- 患者私聊
- 患者群聊
- 人工补充修正

页面、提炼、写回接口都必须围绕这个目标对齐，而不是围绕旧治理台字段组织。

---

## 二、主字段结构

### A. 基本情况
用于识别“这是谁”。

建议字段：
- 患者姓名
- 患者标识
- 性别（如有）
- 年龄 / 年龄段（如有）
- 基础疾病标签 / 主要健康背景（如有）
- 来源渠道（私聊 / 群聊 / 其他）

### B. 喜好习惯
用于支持长期沟通与服务。

建议字段：
- 饮食习惯
- 作息习惯
- 运动习惯
- 沟通偏好
- 明显个人偏好 / 忌讳 / 习惯表达

### C. 核心问题
用于沉淀患者长期与当前的关键问题。

建议字段：
- 长期问题
- 当前问题
- 近期变化
- 主要顾虑 / 阻碍

### D. 历史沟通提炼
用于记录从多轮私聊 / 群聊里持续抽出的关键信息。

建议字段：
- 近期互动摘要
- 历史关键信息摘要
- 已确认事实
- 待确认信息

### E. 跟进重点
用于支撑下一步动作，而不是只记录一次建议。

建议字段：
- 当前跟进重点
- 建议下一步
- 当前优先级
- 是否需要人工重点关注

### F. 档案更新时间
用于判断档案是否陈旧。

建议字段：
- 最近提炼时间
- 最近人工修正时间
- 最近来源会话

---

## 三、页面三块与档案模型的关系

### 1. 患者档案
展示已经沉淀进档案的稳定内容：
- 基本情况
- 喜好习惯
- 核心问题
- 历史沟通提炼
- 跟进重点
- 档案更新时间

### 2. 本次提炼结果
展示本轮新增信息：
- 本次新增基本信息
- 本次新增喜好习惯
- 本次新增问题 / 顾虑
- 本次新增沟通线索
- 本次新增跟进建议

### 3. 人工修正与保存
展示人工确认后的写回动作：
- 对上述字段逐项修正
- 明确哪些写入档案
- 明确哪些暂不写入
- 保存后更新档案更新时间

---

## 四、当前纠偏结论

当前页面里仅保留“需求 / 顾虑 / 跟进点”的结构是不够的。

后续必须从“最小试跑字段模型”升级到“目标对齐档案模型”，否则会出现：
- 页面能试跑，但无法沉淀真正可用的患者档案
- 私聊 / 群聊信息无法被系统性整理
- 后续档案写回缺乏稳定字段结构

---

## 五、后续改造顺序

1. 先按本模型重排患者档案区字段
2. 再按本模型重排本次提炼结果区字段
3. 再按本模型改人工修正与保存区
4. 最后补真实档案写回接口
