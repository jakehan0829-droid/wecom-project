# 详情页继续增强：业务化 patient 候选 / audit 变化表达 / 消息最小筛选 v3

更新时间：2026-03-30

## 本轮新增

### 1. patient 候选更业务化
- 兼容 `/api/v1/patients` 返回 `{ items, total }`
- 候选卡片增加：
  - 手机
  - 性别 / 糖尿病类型
  - 风险等级 / 管理状态
  - 来源

### 2. audit 面板增加变化表达
- 最新 audit 卡片增加“患者变化”表达：`fromPatientId -> toPatientId`
- 增加绑定变化展示
- 审计结果更适合快速判断本次治理到底改了什么

### 3. 消息区最小筛选
- 支持类型筛选：全部 / 文本 / 图片 / 文件
- 支持条数切换：最近 5 条 / 最近 10 条
- 配合已有排序和展开收起，消息区可读性进一步提高

## 补充修正
- 修复 patient 列表前端对后端 `{ items, total }` 结构的兼容问题，避免候选区空白。

详情页已经开始具备“更像真实业务治理台”的判断效率：patient 候选更有上下文，audit 能看变化，消息能筛选。
