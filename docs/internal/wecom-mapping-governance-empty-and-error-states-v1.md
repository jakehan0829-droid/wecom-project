# 企微映射治理台空状态 / 异常状态约定 v1

更新时间：2026-03-30

## 1. 目的

本文件用于明确治理台接口在“没数据”与“异常”时的解释口径，避免前端误渲染，也避免执行人误判。

---

## 2. 空状态是正常的场景

### 2.1 `tables.latestUnmappedCustomers = []`
表示：
- 当前没有识别到未映射私聊对象

前端建议：
- 显示“当前没有未映射对象”
- 不要显示报错态

### 2.2 `tables.latestConflictCustomers = []`
表示：
- 当前没有识别到映射冲突对象

前端建议：
- 显示“当前没有冲突对象”
- 不要显示报错态

### 2.3 `recentActions = []`
常见原因：
- 当前时间窗口内没有治理动作
- 如 `timePreset=yesterday` 但昨天没有操作

前端建议：
- 显示“当前筛选范围内暂无治理动作”

### 2.4 `cards.totalActions = 0`
表示：
- 当前筛选窗口内没有治理动作

这不等于系统坏了。

---

## 3. 哪些空结果要谨慎解释

### 3.1 cards 为 0，但问题对象列表非空
解释：
- 当前窗口内没有新治理动作
- 但系统里仍存在未处理问题对象

这是一种正常业务状态，不是接口异常。

### 3.2 charts 为空，但 tables 有数据
一般不应长期出现。

若出现，优先检查：
- summary 聚合逻辑是否被过滤条件过度收窄
- recentActions 是否来自不同过滤口径

---

## 4. 异常状态判断

### 4.1 HTTP 500
通常表示：
- SQL 异常
- 字段缺失
- 迁移未完成但代码误走稳定列

执行人建议：
1. 看 pm2 logs
2. 看是否出现 `column does not exist`
3. 看是否是 schema owner/迁移问题

### 4.2 返回结构缺字段
如果接口成功但缺少：
- `meta`
- `cards`
- `charts`
- `tables`

应视为接口契约异常，而不是空状态。

### 4.3 某字段为 `null` 但接口整体成功
例如：
- `meta.timePreset = null`

通常表示：
- 当前调用未传该参数
- 不是异常

---

## 5. 前端默认渲染建议

### 成功 + 空数据
- 用 empty state 文案
- 不要红色报错

### 失败 + error.code 存在
- 用 error state
- 可提示“请联系执行人查看 backend 日志”

### 时间范围无数据
- 优先提示“当前筛选范围内暂无数据”
- 而不是“接口异常”

---

## 6. 一句话结论

在治理台里，空数组和 0 计数经常是正常业务状态；真正该当异常处理的，是 HTTP 500、字段缺失、或明显的结构契约破坏。
