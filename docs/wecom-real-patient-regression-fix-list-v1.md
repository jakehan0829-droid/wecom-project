## 剩余细小偏差

### 1. 患者列表字段仍偏弱
当前列表主要还是从 `byConversation` 聚合结果取值，真实数据下：
- 患者姓名可能退化为会话名
- 患者标识可能退化为 customerId / primaryCustomerId

需继续优化：
- 优先补真实患者名
- 区分“患者标识”和“会话标识”

### 2. 患者档案区当前状态判断仍较粗
当前状态判断仍是前端基于是否有风险/需求的简化推断：
- 需重点跟进
- 持续跟进中
- 待补充判断

需继续优化：
- 后续尽量基于更稳定业务字段给状态
- 当前先保留最小规则，但可微调文案

### 3. 本次提炼结果在“无内容”场景下可读性一般
真实数据下当前样本属于“客户表达不足”：
- 结果虽然不空
- 但展示仍容易显得信息弱

需继续优化：
- 无新增时给更自然的空态文案
- 避免让用户误以为出错

### 4. 人工修正区目前是“可编辑字段”，但还不是“真正写回档案字段”
当前已实现逐项编辑，但保存动作仍走旧 feedback 接口语义映射。

需继续优化：
- 后续补真正的档案字段写回接口
- 当前阶段先保留为可验证闭环
