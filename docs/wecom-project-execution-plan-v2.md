# 企微项目详细推进清单 V2（可执行版）

## 1. 文档目的

本文件作为企微项目推进的核心执行清单，统一回答以下问题：
- 项目到底要做什么
- 当前已完成什么
- 还缺什么
- 下一步先做什么
- 如何判断任务是否完成

本清单与《企微项目能力边界 V2》及《企微消息采集与会话模型设计 V1》配套使用，构成企微项目当前阶段的核心推进文件。

---

## 2. 项目总目标

将企微项目从“已打通企业微信发送链路”的阶段，推进为一个：
- 能接入企微
- 能主动触达客户
- 能记录客户群聊与私聊互动
- 能分析客户需求与风险
- 能反哺客户方案与跟进策略
- 能稳定运行和持续增强

的业务系统。

---

## 3. 当前基线

### 3.1 已完成
- 企业微信可信域名校验已通过
- backend 已可运行并由 PM2 托管
- 真实发送已打通
- action 状态回写已具备
- delivery log 已落库
- 支持日志列表、按 actionId 查询、单条详情查询
- dashboard 已具备发送统计
- backend env 边界已收口
- 运行目录 `project/logs/`、`project/ops/`、`project/runtime/` 已建立
- 基础运行/运维文档已补齐一轮

### 3.2 当前关键缺口
- 群聊/私聊消息接收与记录未形成正式主链
- 客户聊天分析能力未形成正式主链
- 方案反哺与跟进建议能力未形成正式主链
- 自动触发器与主动汇报机制未完全打通

---

## 4. 模块拆分与优先级

### 模块 M1：基础接入与运行层
目标：稳定运行、边界清晰、可部署可维护
优先级：P1（持续维护）

### 模块 M2：主动触达与发送执行层
目标：稳定发送、发送结果可查、可排障
优先级：P1（短期核心）

### 模块 M3：客户会话采集与记录层
目标：把群聊和私聊消息纳入系统主链
优先级：P0（必须立刻进入主线）

### 模块 M4：客户聊天分析与需求提炼层
目标：把聊天信息转化为结构化业务判断
优先级：P0（必须立刻进入主线）

### 模块 M5：客户方案反哺与跟进决策层
目标：让分析结果真正影响方案和后续动作
优先级：P0-P1（核心业务价值层）

### 模块 M6：观察、巡检、自动触发与运营层
目标：让系统可观察、可提醒、可主动暴露重点信息
优先级：P1（支撑长期可用）

## 8. 验收口径

### 短期验收口径
- 项目能力边界不再跑偏
- 消息采集与会话模型正式固化
- 发送侧继续增强并可排障

### 中期验收口径
- 至少形成“消息进入 → 入库 → 分析 → 建议”的最小闭环
- 群聊/私聊消息不再只是外部信息流，而成为系统资产

### 长期验收口径
- 项目不只是企微消息发送工具
- 而是一个能吸收客户互动、分析需求变化、反哺方案与跟进动作的业务系统

---

## 9. 文档结论

企微项目 V2 阶段的核心判断是：

> 现阶段不能再把项目仅理解为发送链路建设；
> 必须同步建设客户聊天记录、会话分析、方案反哺与自动触发能力，
> 才能满足真实业务需求与项目预期。
