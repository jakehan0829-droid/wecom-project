# 企微项目多Agent分工与冲刺推进表 V1

## 1. 文档目的

本文件用于正式明确：
- 当前阶段企微项目是唯一前置主项目
- 多Agent系统在当前阶段只作为企微项目的执行与协作工具，不再与企微项目并列抢主线资源
- 各 Agent 在企微项目中的职责、协作边界、交付物和冲刺节奏

目标是解决以下三个问题：
1. 避免企微项目与多Agent系统优化混在一起导致错乱
2. 避免对企微项目核心需求理解偏差导致跑偏
3. 避免推进节奏过慢，无法尽快完成企微项目

---

## 2. 当前阶段总原则

### 2.1 当前唯一前置主项目
**企业微信项目**。

说明：
- 当前阶段所有高优先级推进动作，都优先服务企微项目
- 多Agent 系统不再作为并列主项目展开优化
- 多Agent 当前只承担“协助企微项目加速推进”的角色

### 2.2 企微项目核心需求五条主线
后续所有任务必须服务以下五条核心需求：
1. 企微接入与可运行
2. 企业微信群与私聊消息记录
3. 客户聊天分析
4. 客户方案与需求反哺
5. 业务推进与运营闭环

任何不能直接服务这五条主线的动作，当前阶段都应视为低优先级或偏航风险。

---

## 3. 多Agent 在企微项目中的总分工

## 3.1 主Agent
### 职责
- 项目总控
- 优先级判断
- 任务拆解与路由
- 跨角色收口
- 风险把关
- 最终汇报

### 不再承担
- 长时间单独包办所有角色的细节工作
- 让企微项目停留在纯规划层

### 输出物
- 项目主线判断
- 冲刺节奏控制
- 最终阶段汇报

---

## 3.2 A1 工作秘书
### 职责
- 维护企微项目推进板
- 维护任务状态
- 维护交付清单
- 整理待办 / 阻塞 / 验收项
- 整理定时汇报素材

### 输出物
- 企微项目进度板
- 企微项目任务状态表
- 交付索引与验收清单

### 边界
- 不负责核心技术实现判断
- 不替代主Agent做架构与优先级决策

---

## 3.3 A2 开发工程师
### 职责
- backend 最小实现方案
- intake 接口实现拆解
- analyze 接口实现拆解
- routes / controller / service / SQL 对接方案
- 输入输出样例落地为开发任务

### 输出物
- backend 实现任务单
- 模块实现顺序
- API / service / SQL 实现建议
- 最小闭环实现路径

### 当前阶段重要性
- 当前阶段为企微项目第一执行主力

---

## 3.4 A3 部署运维官
### 职责
- SQL 落地方式评估
- 数据库变更风险检查
- 运行边界与环境风险检查
- PM2 / 服务运行与部署衔接检查
- 上线顺序与回滚建议

### 输出物
- 落库与部署风险说明
- 上线顺序建议
- 回滚预案
- 环境/运行衔接检查单

### 当前阶段重要性
- 在企微项目从设计走向 backend 实现时，必须介入复核

---

## 3.5 A4 策划咨询顾问
### 职责
- 客户互动业务结构梳理
- 聊天分析维度合理性设计
- 客户方案反哺逻辑
- 跟进建议逻辑
- 客户视图 / 方案视图建议

### 输出物
- 分析结构合理性复核
- 方案反哺逻辑说明
- 客户视图与建议视图结构建议

### 当前阶段重要性
- 保证企微项目不退化为单纯消息系统

---

## 3.6 A5 研究助理
### 职责
- 企微消息能力边界调研
- 群聊 / 私聊可行方案调研
- 外部最佳实践参考
- 对标会话 intelligence / CRM 类方案

### 输出物
- 关键能力调研结论
- 外部方案参考摘要

### 当前阶段重要性
- 非默认常驻介入；当企微能力边界或实现路径不清时介入

---

## 3.7 A6 质量审查官
### 职责
- 审核设计是否跑偏
- 审核任务是否过散
- 审核验收标准是否清楚
- 审核高风险实现方案的质量与边界

### 输出物
- 实现方案复核意见
- 验收标准复核意见
- 风险清单

### 当前阶段重要性
- 在关键实现方案形成后必须介入复核

---

## 4. 三段冲刺推进表

## 冲刺 1：最小消息入库闭环
### 目标
让企微客户消息正式进入系统并可回看。

### 关键交付
- 数据表/SQL 草案与 backend 对接路径
- 消息 intake 接口实现任务
- conversation 归档逻辑任务
- customer 关联逻辑任务
- 最小查询回看任务

### 核心验收标准
- 至少一条私聊文本消息可入库并按 customer / conversation 回看

### 分工
- 主Agent：总控与收口
- A2：实现任务拆解主责
- A3：数据库与运行风险复核
- A1：任务状态与验收清单维护

---

## 冲刺 2：最小聊天分析闭环
### 目标
让系统基于聊天记录生成结构化 insight 并可查询。

### 关键交付
- analyze 接口实现任务
- insight 入库任务
- 最小八类输出实现任务
- next_action_suggestion / plan_update_suggestion 生成任务
- 分析结果查询回看任务

### 核心验收标准
- 至少一组聊天可生成结构化 insight 并查回结果

### 分工
- 主Agent：总控与收口
- A2：实现任务拆解主责
- A4：分析结构与业务逻辑复核
- A6：验收标准与质量复核
- A1：进度整理与汇报支撑

---

## 冲刺 3：方案反哺与业务接入
### 目标
让聊天分析结果进入客户方案、跟进动作与提醒体系。

### 关键交付
- 客户方案更新建议落点
- 跟进动作建议落点
- 客户视图 / 方案视图建议
- 高风险对话 / 新增需求 / 重点客户异动提醒最小方案

### 核心验收标准
- insight 不再只是存档，而开始影响后续动作与方案

### 分工
- 主Agent：总控与收口
- A4：业务结构主责
- A2：实现接入支撑
- A1：推进板与验收整理
- A6：关键方案复核

---

## 5. 当前阶段执行规则

### 5.1 不再并列推进多Agent系统优化
除非直接阻碍企微项目，否则多Agent系统优化不再占用主线资源。

### 5.2 只围绕企微项目五条核心需求推进
后续所有任务都必须可映射到五条主线之一。

### 5.3 优先拿最小闭环结果
先打通：
1. 消息入库闭环
2. 聊天分析闭环
3. 方案反哺闭环

### 5.4 汇报只围绕企微项目主线
后续阶段性汇报默认只报：
- 企微项目当前状态
- 已完成
- 进行中
- 风险/阻塞
- 下一步

## 7. 文档结论

本文件的核心结论是：

> 当前阶段企微项目是唯一前置主项目，多Agent系统只作为其执行协作工具存在；
> 后续必须以“主Agent总控 + A2/A3/A4/A1/A6 分工协作 + 三段冲刺推进”的方式尽快拿到企微项目实际结果。
