# 详情页继续增强：真实主客户映射 / patient 匹配理由 / audit 状态轨迹 v5

更新时间：2026-03-30

## 本轮新增

### 1. 当前主客户改为真实字段映射
- conversation detail 真实读取：
  - `primary_customer_id`
  - `mapping_status`
  - `mapping_matched_by`
- 前端不再用占位 patientId 代替“当前主客户”展示

### 2. patient 候选增加匹配理由
- 候选卡片可展示匹配理由，例如：
  - 与当前主客户一致
  - 与当前映射 patient 一致
  - 命中方式参考
  - 最近会话命中
- 帮助治理人员更快判断为什么这个候选更相关

### 3. audit 总结条升级为状态变化轨迹
- 在原有动作总结基础上增加：
  - 治理前状态
  - 治理后状态
- 典型表达：
  - 已从 A 改到 B，状态 X -> Y
  - 已提升为 binding，状态 X -> Y
  - 已撤销人工确认，状态 X -> Y

## 一句话结论

详情页进一步从“能展示结果”推进到“能解释结果”：当前主客户更准、patient 候选更有理由、audit 能表达状态轨迹。
