# 工作台 V1 试跑入口与引导 v1

更新时间：2026-03-31

## 1. 稳定试跑入口

当前推荐统一使用同域入口：

- `http://<服务器地址>:3000/?mode=real`

不要再优先使用：
- 5173 临时 dev 端口
- 127.0.0.1 本机地址（仅服务器本机可访问）

原因：
- 3000 已承载前端静态页面 + 后端 API
- 更适合作为稳定试跑入口

---

## 2. Token 说明

页面中的 Bearer Token：
- 是后端接口访问令牌
- 不是大模型秘钥
- 当前项目使用 JWT 校验

试跑时：
- 切换到 real 模式
- 填入可用 Bearer Token
- 再进入会话详情页试跑

---

## 3. 标准试跑步骤

0. 优先执行一键试跑脚本：

```bash
cd /root/.openclaw/workspace/project/backend
./scripts/open-workbench-v1-trial-run.sh
```

1. 打开脚本输出的标准会话地址
2. 把脚本输出的 accessToken 填入页面 Bearer Token 输入框
3. 查看顶部入口摘要
4. 查看右侧判断层
5. 在“动作与反馈”区完成一次最小反馈
6. 查看本次反馈回流
7. 查看 action history / 证据链

---

## 4. 空态说明

如果页面提示：
- “当前没有待处理动作”

含义是：
- 当前会话没有 pending action
- 不是页面坏了
- 此时无法测试最小反馈闭环

处理方式：
- 切换到有 pending action 的会话
- 或预置一条标准测试动作

---

## 5. 标准测试样本建议

建议后续固定一条标准试跑样本，满足：
- 能稳定进入详情页
- 能看到判断层
- 能看到至少 1 条 pending action
- 能做一次最小反馈提交
- 能看到回流结果

这样后续演示不再需要临时补数。
