## 五、当前系统还存在的边界 / 风险

### 1. 真实企微消息还没打通
当前触达动作仍停留在系统内部模型层，不是实际企微消息发送。

### 2. 企微真实身份链路还没打通
当前绑定仍是 MVP 最小绑定模型。

### 3. 状态机仍是最小闭环
还不是完整 workflow / BPM。

### 4. 监督兜底机制仍以文档规则为主
已解决“无规则”问题，但还未实现自动触发型接管。

### 5. 主Agent 仍然存在模型额度 / token 单点风险
虽然已有接管规则，但自动化切换尚未落地。
